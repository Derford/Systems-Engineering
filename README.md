# Systems-Engineering
=======
# Маркетплейс вакансий и стажировок для университета

## Описание проекта

Проект выполнен в рамках дисциплины **«Системная инженерия»**.

Цель работы — спроектировать информационную систему для публикации вакансий и стажировок внутри университета. Система объединяет студентов, работодателей, карьерный центр и модераторов в единую цифровую среду. В проекте рассмотрены границы системы, требования, архитектура, бизнес-процессы, модели данных, интеграции и сценарии тестирования.

С точки зрения предметной области проект относится к направлению **EdTech / HRTech**. Отдельное внимание уделено рекомендательному модулю, приватности данных студентов, модерации вакансий и аналитике трудоустройства.

## Основные возможности системы

### Для студента

- просмотр вакансий и стажировок;
- получение рекомендованных вакансий по профилю и навыкам;
- отправка отклика на вакансию;
- отслеживание статуса отклика;
- получение уведомлений об интервью и изменениях статуса.

### Для работодателя

- создание и публикация вакансий;
- просмотр откликов студентов;
- работа со статусами кандидатов;
- назначение интервью;
- получение уведомлений о новых откликах.

### Для карьерного центра

- контроль процесса трудоустройства;
- просмотр аналитики по вакансиям и откликам;
- получение отчётов;
- взаимодействие с работодателями;
- оценка активности студентов и компаний.

### Для модератора

- проверка вакансий перед публикацией;
- выявление спама и некорректных предложений;
- блокировка нарушающих работодателей;
- контроль качества контента.

## Установка и запуск проекта

Проект представляет собой документацию в формате **mdBook**.

Для локального запуска потребуются:

- Rust;
- Cargo;
- mdBook;
- Visual Studio Code или другой редактор кода.

### 1. Установка Rust

Скачать установщик Rust можно с официального сайта:

```text
https://rustup.rs/
```

После установки необходимо перезапустить терминал или Visual Studio Code.

Проверка установки:

```bash
cargo --version
```

Если команда выводит версию, Rust установлен корректно.

### 2. Установка mdBook

В терминале выполнить команду:

```bash
cargo install mdbook
```

Проверка установки:

```bash
mdbook --version
```

### 3. Открытие проекта

Распаковать архив проекта, открыть папку в Visual Studio Code:

```text
File → Open Folder
```

Выбрать папку, в которой находятся файлы:

```text
book.toml
README.md
src/
```

### 4. Запуск локального сервера

В терминале перейти в папку проекта:

```bash
cd путь_к_проекту
```

Пример:

```powershell
cd "C:\Users\Rapha\OneDrive\Рабочий стол\career-marketplace-plantuml-final"
```

Запустить mdBook:

```bash
mdbook serve
```

После запуска документация будет доступна в браузере:

```text
http://localhost:3000
```

### 5. Остановка сервера

Для остановки локального сервера используется сочетание клавиш:

```text
Ctrl + C
```

## Диаграммы

Основные страницы документации используют **готовые SVG-изображения**, поэтому диаграммы отображаются в mdBook без дополнительных плагинов.

Финальные диаграммы размещены в папке:

```text
src/diagrams/final/
```

Структура диаграмм:

```text
src/diagrams/final/
├── c4/
├── uml/
├── idef0/
├── idef3/
├── dfd/
├── bpmn/
└── architecture/
```

Исходные материалы диаграмм оставлены отдельно, чтобы при необходимости можно было проверить логику модели или внести изменения.

## Swagger / OpenAPI

Описание API находится в разделе:

```text
src/api/
```

В проекте описаны основные эндпоинты для вакансий, откликов, рекомендаций, интервью и отчётов карьерного центра.

## ADR

Архитектурные решения оформлены в разделе:

```text
src/adr/
```

ADR фиксируют ключевые решения по архитектуре, хранению данных, выделению сервиса рекомендаций, уведомлениям, безопасности и публикации проекта.

## Онлайн-публикация

Проект подготовлен к публикации через **GitHub Pages**.

Для публикации необходимо:

1. Создать репозиторий на GitHub.
2. Загрузить проект в репозиторий.
3. Перейти в `Settings → Pages`.
4. Выбрать источник публикации `GitHub Actions`.
5. Дождаться успешной сборки workflow.

После публикации сайт будет доступен по адресу вида:

```text
https://username.github.io/repository-name/
```
