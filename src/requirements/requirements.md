# Функциональные и нефункциональные требования

## Функциональные требования

| ID | Требование |
|---|---|
| FR-01 | Студент может войти через SSO университета. |
| FR-02 | Студент может заполнить профиль: навыки, направление, резюме, опыт. |
| FR-03 | Работодатель может создать вакансию или стажировку. |
| FR-04 | Модератор может одобрить, отклонить или отправить вакансию на доработку. |
| FR-05 | Студент может просматривать каталог вакансий. |
| FR-06 | Система может рассчитывать рекомендации вакансий. |
| FR-07 | Студент может отправить отклик на вакансию. |
| FR-08 | Работодатель может изменить статус отклика. |
| FR-09 | Работодатель может назначить интервью. |
| FR-10 | Система отправляет уведомления о важных изменениях. |
| FR-11 | Карьерный центр может смотреть отчёты по откликам и трудоустройству. |
| FR-12 | Система ведёт журнал аудита действий. |

## Нефункциональные требования

| ID | Требование | Целевое значение |
|---|---|---|
| NFR-01 | Доступность | не ниже 99.5% в учебной модели |
| NFR-02 | Время ответа API | p95 менее 800 мс для типовых запросов |
| NFR-03 | Каталог вакансий | поиск и фильтрация менее 1 секунды |
| NFR-04 | Приватность | доступ к персональным данным только по ролям |
| NFR-05 | Безопасность | JWT/OAuth2, HTTPS, аудит действий |
| NFR-06 | Модерация | подозрительные вакансии не публикуются без проверки |
| NFR-07 | Качество рекомендаций | Precision@5 не ниже 0.70 на тестовом наборе |
| NFR-08 | Масштабируемость | до 10 000 студентов и 1 000 работодателей |

## Роли доступа

| Роль | Доступ |
|---|---|
| Студент | профиль, каталог, рекомендации, свои отклики |
| Работодатель | свои вакансии, отклики на них, интервью |
| Модератор | очередь проверки и решения по вакансиям |
| Карьерный центр | отчёты, агрегированная аналитика, контроль качества |
| Администратор | настройки, роли, аудит, мониторинг |
