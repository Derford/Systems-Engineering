# CONOPS и OPSCON

## CONOPS

Пользовательская концепция работы системы строится вокруг четырёх основных сценариев.

1. Работодатель создаёт вакансию и отправляет её на модерацию.
2. Модератор проверяет вакансию и публикует её в каталоге.
3. Студент получает рекомендации, открывает вакансию и отправляет отклик.
4. Работодатель рассматривает отклик, назначает интервью и меняет статус кандидата.

## OPSCON

Система работает как веб-платформа с постоянным доступом для студентов и работодателей. Карьерный центр использует административный интерфейс для контроля вакансий и отчётности.

## Нормальный режим

- авторизация через SSO;
- доступ к каталогу вакансий;
- обработка откликов;
- расчёт рекомендаций;
- уведомления по e-mail/мессенджеру;
- запись статусов и действий в аудит.

## Деградационный режим

| Проблема | Поведение системы |
|---|---|
| Недоступен сервис рекомендаций | показывается обычный каталог вакансий |
| Недоступен e-mail | уведомление ставится в очередь повторной отправки |
| Недоступен календарь | интервью сохраняется без синхронизации, пользователь получает предупреждение |
| Недоступна аналитика | основные действия продолжают работать, отчёты формируются позже |

## Критичные данные

- персональные данные студента;
- резюме и навыки;
- отклики и статусы;
- история модерации;
- события аудита.
